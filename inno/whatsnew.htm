My Inno Setup Extensions 3.0.2-beta (2002-06-04)

Changes:
  • Based on Inno Setup 3.0.2-beta
  • Based on Innerfuse Pascal Script 3 (ifps3 CVS module checked out on 2002-06-01)

  • Replaced installer and uninstaller icons by modern looking ones. Did not include any 48x48 or 24 bit icons (for XP) because of their size.
  • Added [Code] section event function CheckSerial and constant {userinfoserial} for initial serial number support. If a CheckSerial event function is present in your [Code] section, a serial number field with automatically appear on the User Info wizard page (which must be enabled using UserInfoPage=yes in your [Setup] section!). Return True to accept the serial number and False to reject it. CheckSerial prototype:

    function CheckSerial(Serial: String): Boolean;

Changes inherited from Inno Setup 3.0.1-beta:

  • Inno Setup 3.0 Revision History

Changes inherited from Innerfuse Pascal Script 3:

  • Added support for record (struct) passing to and from .dlls. Limited to packed (non aligned) records and passing by reference (not really a limitation since this basically covers 99% of all cases).
  • Added 'array constants' which for example makes this possible:

    A := ['foo', 'bar'];

  • Added string indexing support which for example makes this possible:

    S := 'foebar';
    S[3] := 'o';
  • Various fixes.

 
My Inno Setup Extensions 3.0.1-beta (2002-05-09)

Changes:
  • Based on Inno Setup 3.0.1-beta
  • Based on Innerfuse Pascal Script 3 (ifps3 CVS module checked out on 2002-05-09)

  • The [Code] section is now compiled by a 'real' compiler integrated into the My Inno Setup Extensions compiler IDE instead of being interpreted by Setup as in ISX 2. This is possible because of the great work Carlo Kok did on a new version of the scripting engine used by ISX: Innerfuse Pascal Script 3. See www.carlo-kok.com for more information on IFPS3. Thanks Carlo!

    This means:
    • [Code] section errors such a syntax errors will pop up in the compiler IDE instead of in Setup at runtime.
    • The compiler IDE now checks the prototype of event functions, check functions and code constant functions for correctness and will pop up an error if something is wrong.
    • The compiler IDE now checks the existance of check functions and code constant functions and will pop up an error if a function is missing.
    • A compiled Setup no longer contains the orginal [Code] section. Previously it did (although compressed into a non human readable form).

  • The [Code] section debugger has been integrated into the compiler IDE and the new Inno Setup 3 'debugger'. Thanks to Jordan for his help on this.

    This means:
    • [Setup] section directive WizardDebug has been removed. You no longer need to recompile to turn debugging on or off. You can now use the same Setup for debugging and for distributing.
    • You can start 'Run to Cursor' debugging immediately from within the compiler IDE.
    • Since it's integrated into the new IS3 debugger, it shows the same 'green dots' to indicated processed lines, allows you to jump from [Code] section debugging to normal sections debugging seamlessly and offers new debugger features as 'Pause' and tooltip variable evaluation.

  • Because of the above two changes, compiled Setups are now about 20 KB *smaller* then they used to be.

  • #include compiler directive support for the [Code] section. See below for more information on #include.
  • The syntax-highlighting editor now highlights the names of [Code] section event functions.
  • Both {app} and {group} are now available immediately after the user has clicked the Next button on the Select Destination Directory and Select Start Menu Folder wizard pages. Previously they were not available until the installation had started.
  • Add [Code] section support function OutputMsgMemo to show a custom wizard page with a memo. Supports Rich Text. Prototype:

    function OutputMsgMemo(const Prompt, Msg: String): Boolean;

  • Renamed [Code] section support function ChangeDirConst to ExpandConstant.
  • Renamed [Code] section support function ChangeDirConstEx to ExpandConstantEx.
  • Renamed [Code] section support function SelectDir to BrowseForFolder.
  • Modified [Code] section event function UpdateReadyMemo prototype to:

    function UpdateReadyMemo(Space, NewLine, MemoUserInfoInfo, MemoDirInfo, MemoTypeInfo, MemoComponentsInfo, MemoGroupInfo, MemoTasksInfo: String): String;

  • Removed [Code] section support functions UsingWindows4 and UsingNewGUI.
  • Removed [Setup] section directive CodeFile. Use the new #include compiler directive instead.
  • In a code constant such as {code:FunctionName|DefaultValue}, FunctionName may no longer include constants.
  • The syntax of the extern keyword for DLL usage has been modified. See the ScriptDll.iss example file.
  • Minor tweaks and fixes.
